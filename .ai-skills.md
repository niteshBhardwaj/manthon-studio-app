# Manthan Studio — AI Skills Reference

## Project Overview
Manthan Studio is an Electron + React + TypeScript desktop application for AI media generation.
It uses a modular provider architecture to support multiple AI backends (Google Veo 3.1, Nano Banana, Lyria 3, etc.)

## Tech Stack
- **Framework**: `electron-vite` (Electron + Vite + React 19 + TypeScript)
- **Styling**: Tailwind CSS v4 (CSS-first `@theme` directive) + shadcn/ui compatible
- **State**: Zustand (lightweight, TypeScript-first stores)
- **API**: `@google/genai` SDK for Google Gemini API
- **Routing**: TanStack Router (to be added)
- **Icons**: `lucide-react`
- **Animations**: `framer-motion`

## Architecture Patterns

### 1. Process Separation (Electron)
```
src/main/       → Node.js main process (API calls, file I/O, encryption)
src/preload/    → contextBridge (secure IPC bridge)
src/renderer/   → React UI (no Node access)
```

**Key Rule**: Never use `@google/genai` in the renderer. All API calls go through IPC.

### 2. IPC Communication Pattern
```typescript
// Preload: Expose typed API
contextBridge.exposeInMainWorld('manthan', {
  generateVideo: (params) => ipcRenderer.invoke('gen:video', params),
})

// Main: Handle IPC
ipcMain.handle('gen:video', async (_event, params) => {
  const provider = providerRegistry.getActive()
  return provider.generateVideo(params)
})

// Renderer: Use the API
const result = await window.manthan.generateVideo({ prompt: '...' })
```

### 3. Provider System
Each AI provider implements the `MediaProvider` interface:
```typescript
interface MediaProvider {
  id: string
  name: string
  supportedModalities: Modality[]
  initialize(apiKey: string): Promise<void>
  testConnection(): Promise<ConnectionStatus>
  generateVideo?(params: VideoGenParams): Promise<GenerationOperation>
  generateImage?(params: ImageGenParams): Promise<GenerationResult>
  pollOperation?(operationId: string): Promise<GenerationOperation>
}
```

Providers are registered in `ProviderRegistry` (singleton). New providers just implement this interface.

### 4. Encrypted Key Storage
- Uses `electron.safeStorage` (Windows DPAPI / macOS Keychain)
- Keys stored as encrypted base64 in a JSON file
- Located at `{userData}/manthan-keys.json`

### 5. State Management (Zustand)
Three stores:
- `useAppStore` — UI state (sidebar, modals, search)
- `useGenerationStore` — Prompt state, jobs queue, frame inputs
- `useProviderStore` — Provider list, active provider, connection status

### 6. Design System
- **Colors**: OKLCH color space for perceptual uniformity
- **Dark theme**: `oklch(0.15 0.01 260)` base
- **Glassmorphism**: `backdrop-filter: blur(20px)`
- **Font**: Inter from Google Fonts
- **Animations**: Framer Motion for layout transitions

## Key APIs

### Google Veo 3.1 (Video)
```typescript
const ai = new GoogleGenAI({ apiKey })
// Text → Video
const op = await ai.models.generateVideos({
  model: 'veo-3.1-generate-preview',
  prompt: '...',
  config: { aspectRatio: '16:9', resolution: '1080p' }
})
// Poll until done
while (!op.done) {
  op = await ai.operations.getVideosOperation({ operation: op })
}
```

### Google Nano Banana (Image)
```typescript
const response = await ai.models.generateContent({
  model: 'gemini-3.1-flash-image-preview',
  contents: 'Create an image...'
})
// Extract image from response.candidates[0].content.parts
```

## Common Tasks

### Adding a new provider
1. Create `src/main/providers/new-provider.ts` implementing `MediaProvider`
2. Register in `src/main/providers/registry.ts`
3. Add meta info in `ApiKeyManager.tsx`

### Adding a new IPC channel
1. Add handler in `src/main/ipc/index.ts`
2. Expose in `src/preload/index.ts`
3. Add types in `src/preload/index.d.ts`
4. Use `window.manthan.newMethod()` in renderer

### Changing the design system
Edit `src/renderer/src/globals.css` — all tokens are in the `@theme inline` block.

## Known Issues / Gotchas
- `electron-store` v11+ is ESM-only — replaced with custom `JsonStore` class
- DevTools Autofill.enable errors are harmless Electron/Chromium noise
- `safeStorage` may not be available in dev mode — fallback to base64 encoding
- Video generation is async — must poll `getVideosOperation()` every ~10 seconds
